## You Might Not Need a Framework

Frameworks require overhead of both kbs and onboarding.

Remember that bootstrap or foundation code?

Write only what you need, use variables as the backbone.

Always use variables for any color, typography, spacing and your entire site can be updated or configured in one fell swoop. Don't worry about creating unique components if they are all using custom properties.

<Notes>
  Making your own might seem overwhelming, BUT you don't need to making
	something gigantic to make it effective.

    We all know how it feels to look at code you didn't write

</Notes>
